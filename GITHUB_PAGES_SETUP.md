# Настройка GitHub Pages для Google Analytics

## Проблема
Ваш сайт ausfinanz.github.io показывает 404 ошибку, что означает GitHub Pages не настроен.

## Решение: Настройка GitHub Pages

### 1. Зайдите в настройки репозитория
1. Откройте [GitHub](https://github.com/ausfinanz/ausfinanz)
2. Нажмите на вкладку "Settings" (Настройки)

### 2. Найдите раздел Pages
1. В левом меню найдите раздел "Pages"
2. Нажмите на него

### 3. Настройте GitHub Pages
1. В разделе "Source" выберите:
   - **Branch**: `main` (или `master`)
   - **Folder**: `/ (root)` - корневая папка
2. Нажмите "Save"

### 4. Подождите активации
- GitHub Pages активируется в течение 5-10 минут
- Вы увидите сообщение: "Your site is ready at https://ausfinanz.github.io/"

### 5. Проверьте сайт
1. Перейдите по адресу: https://ausfinanz.github.io/
2. Убедитесь, что сайт загружается

### 6. Проверьте Google Analytics
1. Откройте [Google Analytics](https://analytics.google.com/)
2. Перейдите в "Реальное время"
3. Посетите ваш сайт ausfinanz.github.io
4. Убедитесь, что пользователи появляются

## Альтернативные варианты

### Если нет вкладки Settings:
1. Убедитесь, что вы являетесь владельцем репозитория
2. Или попросите владельца настроить GitHub Pages

### Если ветка не main:
1. Проверьте, какая ветка используется в репозитории
2. Выберите соответствующую ветку в настройках Pages

## Проверка после настройки

После настройки GitHub Pages:
1. Подождите 10-15 минут
2. Проверьте, что сайт доступен
3. Проверьте Google Analytics через 24-48 часов

## Если проблемы остаются
1. Убедитесь, что файлы `index.html` и `js/main.js` загружены на GitHub
2. Проверьте, что Google Analytics ID G-Q5PFM8TSB5 соответствует вашему свойству
3. Подождите 24-48 часов для полной индексации Google Analytics
